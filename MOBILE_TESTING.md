# Інструкція для тестування на мобільному телефоні

## Варіант 1: Через веб-інтерфейс (Рекомендовано)

1. **Відкрийте в браузері:**
   ```
   https://stockflow-311.preview.emergentagent.com
   ```

2. **Натисніть кнопку "Connect" або відкрийте DevTools**

3. **Ви побачите QR код для сканування**

4. **На телефоні:**
   - iOS: Відкрийте Camera app і наведіть на QR
   - Android: Встановіть Expo Go і скануйте QR через додаток

## Варіант 2: Пряме підключення

### iOS (iPhone/iPad)

1. **Встановіть Expo Go з App Store**
   - Пошук: "Expo Go"
   - Розробник: 650 Industries

2. **Відкрийте Expo Go**

3. **Введіть URL вручну:**
   ```
   exp://meat-inventory-2.preview.emergentagent.com
   ```

### Android

1. **Встановіть Expo Go з Google Play**
   - Пошук: "Expo Go"
   - Розробник: Expo

2. **Відкрийте Expo Go**

3. **Відскануйте QR код з веб-інтерфейсу**
   - АБО введіть URL вручну:
   ```
   exp://meat-inventory-2.preview.emergentagent.com
   ```

## Варіант 3: Локальне тестування

Якщо ви в одній WiFi мережі з сервером:

1. **Дізнайтеся IP адресу сервера:**
   ```bash
   hostname -I
   ```

2. **На телефоні в Expo Go введіть:**
   ```
   exp://[IP_ADDRESS]:3000
   ```
   Наприклад: `exp://192.168.1.100:3000`

## Перевірка що додаток працює

Після успішного підключення ви побачите:

✅ **Екран "Залишки"** з 4 вкладками внизу:
- Залишки (іконка склад)
- Операції (іконка стрілки)
- Інвентаризація (іконка список)
- Журнал (іконка годинник)

### Тестування функцій:

1. **Залишки:**
   - Має відображати 132 позиції номенклатури
   - Пошук працює
   - Фільтрація за категоріями

2. **Операції:**
   - Можна додати прихід/розхід
   - Вибрати номенклатуру
   - Ввести кількість

3. **Інвентаризація:**
   - Почати повну або часткову інвентаризацію
   - Ввести фактичні залишки

4. **Журнал:**
   - Переглянути історію операцій
   - Синхронізувати offline операції

## Можливі проблеми та рішення

### "Unable to connect to Metro bundler"
**Рішення:** Переконайтеся що:
- Ви в одній мережі з сервером
- Порт 3000 відкритий
- URL введено правильно

### "Network request failed"
**Рішення:** 
- Перевірте інтернет підключення
- Backend має бути доступний на `https://stockflow-311.preview.emergentagent.com/api/health`

### "Could not find navigation object"
**Рішення:** Це вже виправлено, перезапустіть додаток

### Версія React Native не збігається
**Рішення:** Оновіть Expo Go до останньої версії (має бути 0.81.4+)

## Примітки

- Додаток працює в offline режимі
- Операції без інтернету зберігаються локально
- При появі інтернету - автоматична синхронізація
- Всі тексти українською мовою
- Підтримка iOS та Android

## Контакти

Якщо виникли проблеми:
1. Перевірте логи Expo на сервері
2. Перезапустіть Expo: `sudo supervisorctl restart expo`
3. Почистіть кеш: `cd /app/frontend && rm -rf .expo node_modules/.cache`
